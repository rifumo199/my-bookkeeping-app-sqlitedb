Microsoft Visual Studio 2022

Командная строка Visual Studio

Использование: vs_enterprise.exe [command [options]]

(Замените vs_enterprise.exe соответствующим выпуском продукта, который вы устанавливаете.)

Команда командной строки Описание
(blank) Устанавливает продукт.
изменить Изменяет установленный продукт.
обновить Обновляет установленный продукт.
восстановить Восстанавливает установленный продукт.
резюме Возобновляет установку.
экспорт Экспортирует файл конфигурации установки.
modifySettings Изменяет параметры существующей установки.
removeChannel Удаляет канал
updateAll Обновляет все применимые установки.
удалить Удаляет установленный продукт.


Установить параметр командной строки Описание
--installPath <dir> Каталог установки экземпляра, с которым нужно работать. Для команды установки это необязательно, и именно здесь будет установлен экземпляр. Для других команд это требуется, и именно здесь был установлен ранее установленный экземпляр.
--addProductLang <language-locale> Во время установки или изменения это определяет, какие языковые пакеты пользовательского интерфейса установлены в продукте. Это может появиться несколько раз в командной строке для добавления нескольких языковых пакетов. Если это отсутствует, при установке используются региональные настройки машины. См. полный список поддерживаемых языковых стандартов ниже.
--removeProductLang <language-locale> Во время установки или изменения это определяет, какие языковые пакеты пользовательского интерфейса следует удалить из продукта. Это может появиться несколько раз в командной строке для добавления нескольких языковых пакетов. См. полный список поддерживаемых языковых стандартов ниже.
--add <one or more workload or component IDs> Один или несколько идентификаторов рабочей нагрузки или компонента, которые необходимо добавить. Установлены необходимые компоненты артефакта, но не рекомендуемые или дополнительные компоненты. Вы можете глобально управлять дополнительными компонентами, используя --includeRecommended and/or --includeOptional. Чтобы включить несколько рабочих нагрузок или компонентов, повторите --add команду (например, --add Workload1 --add Workload2). Для более точного управления вы можете добавить ;includeRecommended or ;includeOptional к идентификатору (например, --add Workload1;includeRecommended или --add Workload2;includeRecommended;includeOptional). Вы можете повторно использовать эту опцию по мере необходимости.
--remove <one or more workload or component IDs> Один или несколько идентификаторов рабочей нагрузки или компонента для удаления.
--in <path> URI или путь к файлу отклика.
--all Следует ли устанавливать все рабочие нагрузки и компоненты для продукта.
--allWorkloads Устанавливает все рабочие нагрузки и компоненты, без рекомендуемых или дополнительных компонентов.
--includeRecommended Включает рекомендуемые компоненты для любых установленных рабочих нагрузок, но не дополнительные компоненты. Рабочие нагрузки указываются с помощью --allWorkloads или --add.
--includeOptional Включает дополнительные компоненты для всех установленных рабочих нагрузок, но не рекомендуемые компоненты. Рабочие нагрузки указываются с помощью --allWorkloads или --add.
--quiet, -q Не отображать пользовательский интерфейс во время установки.
--passive, -p Отображать пользовательский интерфейс, но не требовать взаимодействия с пользователем.
--norestart Если они присутствуют, команды с --passive или --quiet не будут автоматически перезапускать компьютер (при необходимости). Это игнорируется, если не указан --passive ни --quiet
--nickname <name> Это определяет псевдоним, назначаемый установленному продукту. Псевдоним не может быть длиннее 10 символов.
--productKey <key> Это определяет ключ продукта, который будет использоваться для установленного продукта. Он состоит из 25 буквенно-цифровых символов в формате xxxxx-xxxxx-xxxxx-xxxxx-xxxxx or xxxxxxxxxxxxxxxxxxxxxxxxx.
--config <fullpath> Во время установки или изменения это определяет рабочие нагрузки и компоненты, которые необходимо добавить на основе ранее сохраненного файла конфигурации установки. Эта операция является аддитивной и не удалит рабочую нагрузку или компонент, если они отсутствуют в файле. Также не будут добавлены элементы, которые не относятся к продукту. Во время экспорта это определяет место для сохранения файла конфигурации установки. Это должен быть абсолютный путь.
--help, --?, -h, -? Отображение использования параметра.


Параметр командной строки макета Описание
--layout <dir> Указывает каталог для создания кэша автономной установки.
--lang <one or more language-locales> Используется с --layout для подготовки кэша автономной установки с пакетами ресурсов на указанных языках. См. полный список поддерживаемых языковых стандартов ниже.
--add <one or more workload or component IDs> Один или несколько идентификаторов рабочей нагрузки или компонента, которые необходимо добавить. Установлены обязательные компоненты артефакта, но не рекомендуемые или необязательные компоненты. Вы можете управлять дополнительными компонентами глобально, используя --includeRecommended and/or --includeOptional. Для более тонкого контроля вы можете добавить ;includeRecommended или ;includeOptional к идентификатору (например, --add Workload1;includeRecommended or --add Workload2;includeOptional).
Заметка: Если используется --add, загружаются только указанные рабочие нагрузки и компоненты и их зависимости. Если --add не указан, все нагрузки и компоненты загружаются в макет.
--includeRecommended Включает рекомендуемые компоненты для любых установленных рабочих нагрузок, но не дополнительные компоненты. Рабочие нагрузки указываются с помощью --allWorkloads or --add.
--includeOptional Включает рекомендуемые и дополнительные компоненты для любых рабочих нагрузок, включенных в макет. Рабочие нагрузки указываются с помощью --add.
--keepLayoutVersion Применить изменения к макету без обновления версии макета.
--verify Проверьте содержимое макета. Все поврежденные или отсутствующие файлы перечислены.
--fix--fix Проверьте содержимое макета. Если какие-либо файлы повреждены или отсутствуют, они загрузятся повторно. Для исправления макета требуется доступ в Интернет.
--clean <one or more paths to catalogs> Удаляет старые версии компонентов из макета, который был обновлен до более новой версии.


Расширенный параметр командной строки Описание
--channelId <id> Идентификатор канала для устанавливаемого экземпляра. Это требуется для команды установки и игнорируется для других команд, если указан --installPath is specified.
--channelUri <uri> URI манифеста канала. Если обновления нежелательны, --channelUri может указывать на несуществующий файл. (например, --channelUri C:\doesntExist.chman). Это можно использовать для команды установки; он игнорируется для других команд.
--installChannelUri <uri> URI манифеста канала, который будет использоваться для установки. URI, указанный в --channelUri (который должен быть указан, если указан --installChannelUri), используется для обнаружения обновлений. Это можно использовать для команды установки, для других команд это игнорируется.
--installCatalogUri <uri> URI манифеста каталога, используемый для установки. Если указано, диспетчер каналов пытается загрузить манифест каталога из этого URI перед использованием URI в манифесте канала установки. Этот параметр используется для поддержки автономной установки, при которой кэш макета будет создан с уже загруженным каталогом продуктов. Это можно использовать для команды установки, для других команд это игнорируется.
--productId <id> Идентификатор продукта для устанавливаемого экземпляра. Предварительно заполняется при нормальных условиях установки.
--newChannelUri <uri> Новый URI или путь к манифесту канала, который будет использоваться для обновлений.
--layoutUri <uri> URI размещенного веб-макета, который используется для получения пакетов установки.
--theme <theme> Изменяет цветовую тему установщика. Значения могут быть Light, Dark или HighContrast.
--wait--wait Процесс будет ждать завершения установки, прежде чем вернуть код выхода. Это полезно при автоматизации установки, когда нужно дождаться завершения установки, чтобы обработать код возврата из этой установки.
--locale <language-locale> Измените язык отображения пользовательского интерфейса самой программы установки. Параметр будет сохраняться. См. полный список поддерживаемых языковых стандартов ниже.
--cache Если они присутствуют, пакеты будут сохранены после установки для последующего восстановления. Это переопределяет параметр глобальной политики, который будет использоваться для последующих установок, восстановления или модификации. Политика по умолчанию — кэшировать пакеты. Это игнорируется для команды удаления.
--nocache Если они присутствуют, пакеты будут удалены после установки или восстановления. Они будут повторно загружены только при необходимости и снова удалены после использования. Это переопределяет параметр глобальной политики, который будет использоваться для последующих установок, восстановления или модификации. Политика по умолчанию — кэшировать пакеты.
--noUpdateInstaller Если присутствует, запрещает установщику обновлять себя, когда указано тихо. Установщик не выполнит команду и вернет ненулевой код выхода, если noUpdateInstaller указан с параметром тихо, когда требуется обновление установщика.
--noWeb Если они есть, программа установки Visual Studio использует файлы в каталоге макета для установки Visual Studio. Если пользователь попытается установить компоненты, которых нет в макете, произойдет сбой установки.
--path <name>=<path> Используется для указания пользовательских путей установки. Поддерживаемые имена путей: общий доступ, кэш и установка.
--path cache=<path> Используйте указанное место для загрузки установочных файлов. Это расположение можно задать только при первой установке Visual Studio. Пример: --path cache="C:\VS\cache"
--path shared=<path> Содержит общие файлы для параллельных установок Visual Studio. Некоторые инструменты и пакеты SDK устанавливаются в папку на этом диске, а некоторые другие могут переопределить этот параметр и установить на другой диск. Пример: --path shared="C:\VS\shared"

Важно! Это можно задать только один раз и при первой установке Visual Studio.
--path install=<path> Эквивалентно –-installPath. В частности, --installPath "C:\VS" и --path install="C:\VS" эквивалентны. Одновременно можно использовать только один из них.
