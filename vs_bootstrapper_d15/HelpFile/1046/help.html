Microsoft Visual Studio 2022

linha de comando do Visual Studio

Uso: vs_enterprise.exe [comando [opções]]

(Substitua vs_enterprise.exe conforme apropriado para a edição do produto que você está instalando.)

Comando de linha de comando Descrição
(blank) Instala o produto.
modify Modifica um produto instalado.
update Atualiza um produto instalado.
repair Repara um produto instalado.
resume Retoma a instalação.
export Exporta um arquivo de configuração de instalação.
modifySettings Modifica as configurações de uma instalação existente.
removeChannel Remove um canal.
updateAll Atualiza todas as instalações aplicáveis.
uninstall Desinstala um produto instalado.


Instalar opção de linha de comando Descrição
--installPath <dir> O diretório de instalação do qual a instância atuará. Para o comando de instalação, isso é opcional e é onde a instância será instalada. Para outros comandos, isso é obrigatório e é onde a instância instalada anteriormente foi instalada.
--addProductLang <language-locale> Durante uma operação de instalação ou modificação, isso determina os pacotes de idiomas da interface do usuário instalados no produto. Ele pode aparecer várias vezes na linha de comando para adicionar vários pacotes de idiomas. Se não estiver presente, a instalação usará a localidade do computador. Veja uma lista completa de localidades de idioma com suporte abaixo.
--removeProductLang <language-locale> Durante uma operação de instalação ou modificação, isso determina os pacotes de idiomas da interface do usuário que devem ser removidos do produto. Ele pode aparecer várias vezes na linha de comando para adicionar vários pacotes de idiomas. Veja uma lista completa de localidades de idioma com suporte abaixo.
--add <uma ou mais IDs de carga de trabalho ou componente> Uma ou mais IDs de carga de trabalho ou componente para add. Os componentes necessários do artefato estão instalados, mas não os componentes recomendados ou opcionais. Você pode controlar componentes adicionais globalmente usando --includeRecommended e/ou --includeOptional. Para incluir várias cargas de trabalho ou componentes, repita o comando --add (por exemplo, --add Workload1 --add Workload2). Para um controle mais refinado, você pode acrescentar ;includeRecommended ou ;includeOptional para a ID (por exemplo, --add Workload1;includeRecommended ou --add Workload2;includeRecommended;includeOptional). Você pode repetir essa opção conforme necessário.
--remove <uma ou mais IDs de carga de trabalho ou componente> Uma ou mais IDs de carga de trabalho ou de componente a serem removidas.
--in <path> O URI ou o caminho para um arquivo de resposta.
--all Se todas as cargas de trabalho e componentes de um produto devem ser instalados.
--allWorkloads Instala todas as cargas de trabalho e componentes, sem componentes recomendados ou opcionais.
--includeRecommended Inclui os componentes recomendados para todas as cargas de trabalho instaladas, mas não os componentes opcionais. As cargas de trabalho são especificadas com --allWorkloads ou --add.
--includeOptional Inclui os componentes opcionais para todas as cargas de trabalho instaladas, mas não os componentes recomendados. As cargas de trabalho são especificadas com --allWorkloads ou --add.
--quiet, -q Não exiba nenhuma interface do usuário ao executar a instalação.
--passive, -p Exiba a interface do usuário, mas não solicite nenhuma interação do usuário.
--norestart Se presente, comandos com --passive ou --quiet não reiniciará automaticamente o computador (se necessário). Isso será ignorado se nem --passive nem --quiet forem especificados.
--nickname <name> Isso define o apelido a ser atribuído a um produto instalado. O apelido não pode ter mais de 10 caracteres.
--productKey <key> Isso define a chave do produto (Product Key) a ser usada para um produto instalado. Ele é composto por 25 caracteres alfanuméricos no formato xxxxx-xxxxx-xxxxx-xxxxx-xxxxx-xxxxx ou xxxxxxxxxxxxxxxxxxxxxxxxx.
--config <fullpath> Durante uma operação de instalação ou modificação, isso determina as cargas de trabalho e os componentes para adicionar com base em um arquivo de configuração de instalação salvo anteriormente. Esta operação é aditiva e não removerá nenhuma carga de trabalho ou componente se eles não estiverem presentes no arquivo. Além disso, os itens que não se aplicam ao produto não serão adicionados. Durante uma operação de exportação, determina o local para salvar o arquivo de configuração de instalação. Esse deve ser um caminho absoluto.
--help, --?, -h, -? Exibir o uso do parâmetro.


Opção de linha de comando de layout Descrição
--layout <dir> Especifica um diretório para criar um cache de instalação offline.
--lang <uma ou mais localidades de idioma> Usado com --layout para preparar um cache de instalação offline com pacotes de recursos com o(s) idioma(s) especificado(s). Veja uma lista completa de localidades de idioma com suporte abaixo.
--add <uma ou mais IDs de carga de trabalho ou componente> Uma ou mais IDs de carga de trabalho ou de componente a serem adicionadas. Os componentes necessários do artefato estão instalados, mas não os componentes recomendados ou opcionais. Você pode controlar componentes adicionais globalmente usando --includeRecommended e/ou --includeOptional. Para um controle mais refinado, você pode acrescentar ;includeRecommended ou ;includeOptional para a ID (por exemplo, --add Workload1;includeRecommended ou --add Workload2;includeOptional).
Observação: Se --add é usado, somente as cargas de trabalho e componentes especificados e suas dependências são baixados. Se --add não for especificado, todas as cargas de trabalho e componentes serão baixados para o layout.
--includeRecommended Inclui os componentes recomendados para todas as cargas de trabalho instaladas, mas não os componentes opcionais. As cargas de trabalho são especificadas com --allWorkloads ou --add.
--includeOptional Inclui os componentes opcionais e recomendados para todas as cargas de trabalho incluídas no layout. As cargas de trabalho são especificadas com --add.
--keepLayoutVersion Aplique as alterações ao layout sem atualizar a versão do layout.
--verify Verifique o conteúdo de um layout. Todos os arquivos corrompidos ou ausentes são listados.
--fix Verifique o conteúdo de um layout. Se algum arquivo estiver corrompido ou ausente, eles serão rebaixados. O acesso à Internet é necessário para corrigir um layout.
--clean <um ou mais caminhos para catálogos> Remove versões antigas de componentes de um layout que foi atualizado para uma versão mais recente.


Opção de linha de comando avançada Descrição
--channelId <id> A ID do canal para a instância a ser instalada. Isso é necessário para o comando de instalação, ignorado para outros comandos se --installPath for especificado.
--channelUri <uri> O URI do manifesto do canal. Se as atualizações não forem desejadas, --channelUri pode apontar para um arquivo inexistente. (por exemplo, --channelUri C:\doesntExist.chman) Isso pode ser usado para o comando de instalação; ele é ignorado para outros comandos.
--installChannelUri <uri> O URI do manifesto do canal a ser usado para a instalação. O URI especificado por --channelUri (que deve ser especificado quando --installChannelUri é especificado) é usado para detectar atualizações. Isso pode ser usado para o comando de instalação; ele é ignorado para outros comandos.
--installCatalogUri <uri> O URI do manifesto do catálogo a ser usado para a instalação. Se especificado, O gerenciador de canais tenta baixar o manifesto do catálogo deste URI antes de usar o URI no manifesto do canal de instalação. Esse parâmetro é usado para dar suporte à instalação offline, onde o cache de layout será criado com o catálogo de produtos já baixado. isso pode ser usado para o comando de instalação; ele é ignorado para outros comandos.
--productId <id> A ID do produto para a instância que será instalada. Isso é preenchido previamente em condições normais de instalação.
--newChannelUri <uri> O novo URI ou caminho para o manifesto do canal a ser usado para atualizações.
--layoutUri <uri> O URI de layout hospedado na web que é usado para buscar pacotes de instalação.
--theme <theme> Altera o tema de cor do instalador. Os valores podem ser Light/Dark/HighContrast.
--wait O processo aguardará até que a instalação seja concluída antes de retornar um código de saída. Isso é útil ao automatizar instalações em que é necessário aguardar a conclusão da instalação para manipular o código de retorno dessa instalação.
--locale <language-locale> Altere o idioma de exibição da interface do usuário para o próprio instalador. A configuração será mantida. Veja uma lista completa de localidades de idioma com suporte abaixo.
--cache Se presente, os pacotes serão mantidos após serem instalados para reparos subsequentes. Isso substitui a configuração de política global a ser usada para instalações, reparos ou modificações subsequentes. A política padrão é armazenar pacotes em cache. Isso é ignorado para o comando de desinstalação.
--nocache Se presente, os pacotes serão excluídos após serem instalados ou reparados. Eles serão baixados novamente somente se necessário e excluídos novamente após o uso. Isso substitui a configuração de política global a ser usada para instalações, reparos ou modificações subsequentes. A política padrão é armazenar pacotes em cache.
--noUpdateInstaller Se estiver presente, impedirá que o instalador se atualize quando o modo silencioso for especificado. O instalador falhará no comando e retornará um código de saída diferente de zero se noUpdateInstaller for especificado com quiet quando uma atualização do instalador for necessária.
--noWeb Se estiver presente, Visual Studio instalação usará os arquivos em seu diretório de layout para instalar o Visual Studio. Se um usuário tentar instalar componentes que não estão no layout, a instalação falhará.
--path <name>=<path> Usado para especificar caminhos de instalação personalizados para a instalação. Os nomes de caminho com suporte são compartilhados, armazenados em cache e instalados.
--path cache=<path> Usa o local especificado para baixar arquivos de instalação. Esse local só pode ser definido na primeira vez que Visual Studio é instalado. Exemplo: --path cache="C:\VS\cache"
--path shared=<path> Contém arquivos compartilhados para instalações de Visual Studio lado a lado. Algumas ferramentas e SDKs são instalados em um local nesta unidade, enquanto outras podem substituir essa configuração e instalar em outra unidade. Exemplo: --path shared="C:\VS\shared"

Importante: Isso pode ser definido apenas uma vez e na primeira vez que Visual Studio é instalado.
--path install=<path> Equivalente a –-installPath. Especificamente, --installPath "C:\VS" e --path install="C:\VS" são equivalentes. Apenas um deles pode ser usado por vez.
