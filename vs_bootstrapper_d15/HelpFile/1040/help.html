Microsoft Visual Studio 2022

Riga di comando di Visual Studio

Sintassi: vs_enterprise.exe [comando [opzioni]]

Sostituire vs_enterprise.exe in base all'edizione del prodotto che si sta installando.

Comando da riga di comando Descrizione
(blank) Installa il prodotto.
modify Modifica un prodotto installato.
update Aggiorna un prodotto installato.
repair Ripristina un prodotto installato.
resume Riprende l'installazione.
export Esporta un file di configurazione dell'installazione.
modifySettings Modifica le impostazioni di un'installazione esistente.
removeChannel Rimuove un canale.
updateAll Aggiornare tutte le installazioni applicabili.
uninstall Disinstalla un prodotto installato.


Installare l'opzione della riga di comando Descrizione
--installPath <dir> Directory di installazione dell'istanza su cui intervenire. Per il comando di installazione, è facoltativo ed è la posizione in cui verrà installata l'istanza. Per altri comandi, è obbligatorio ed è la posizione in cui è stata installata l'istanza installata in precedenza.
--addProductLang <language-locale> Durante un'operazione di installazione o modifica, vengono determinati i Language Pack dell'interfaccia utente installati nel prodotto. Può essere visualizzato più volte nella riga di comando per aggiungere più Language Pack. Se non è presente, l'installazione usa le impostazioni locali del computer. Vedere un elenco completo delle impostazioni locali della lingua supportate di seguito.
--removeProductLang <language-locale> Durante un'operazione di installazione o modifica, vengono determinati i Language Pack dell'interfaccia utente da rimuovere dal prodotto. Può essere visualizzato più volte nella riga di comando per aggiungere più Language Pack. Vedere un elenco completo delle impostazioni locali della lingua supportate di seguito.
--add <one or more workload or component IDs> Uno o più ID del carico di lavoro o dei componenti da aggiungere. Vengono installati i componenti necessari dell'artefatto, ma non i componenti consigliati o facoltativi. È possibile controllare componenti aggiuntivi a livello globale usando --includeRecommended e/o --includeOptional. Per includere più carichi di lavoro o componenti, ripetere il comando --add (ad esempio, --add Workload1 --add Workload2). Per un controllo con granularità più fine, è possibile aggiungere ;includeRecommended o ;includeOptional al ID, (ad esempio --add Workload1;includeRecommended o --add Workload2;includeRecommended;includeOptional). È possibile ripetere questa opzione in base alle esigenze.
--remove <one or more workload or component IDs> Uno o più ID del carico di lavoro o del componente da rimuovere.
--in <path> URI o percorso di un file di risposta.
-all Indica se installare tutti i carichi di lavoro e i componenti per un prodotto.
--allWorkloads Installa tutti i carichi di lavoro e i componenti, senza componenti consigliati o facoltativi.
--includeRecommended Include i componenti consigliati per tutti i carichi di lavoro installati, ma non i componenti facoltativi. I carichi di lavoro vengono specificati con --allWorkloads o --add.
--includeOptional Include i componenti facoltativi per tutti i carichi di lavoro installati, ma non i componenti consigliati. I carichi di lavoro vengono specificati con --allWorkloads o --add.
--quiet, -q Non visualizzare alcuna interfaccia utente durante l'installazione.
--passive, -p Visualizza l'interfaccia utente, ma non richiede alcuna interazione da parte dell'utente.
--norestart Se presente, i comandi con --passive o --quiet non riavvieranno automaticamente il computer (se necessario). Viene ignorato se non vengono specificati né --passive né --quiet.
--nickname <name> Definisce il nome alternativo da assegnare a un prodotto installato. Il nome alternativo non può contenere più di 10 caratteri.
--productKey <key> Definisce il codice Product Key da usare per un prodotto installato. È composto da 25 caratteri alfanumerici nel formato xxxxx-xxxxx-xxxxx-xxxxx-xxxxx o xxxxxxxxxxxxxxxxxxxxxxxxx.
--config <fullpath> Durante un'operazione di installazione o modifica, vengono determinati i carichi di lavoro e i componenti da aggiungere in base a un file di configurazione dell'installazione salvato in precedenza. Questa operazione è additiva e non rimuove alcun carico di lavoro o componente se non sono presenti nel file. Inoltre, gli elementi che non si applicano al prodotto non verranno aggiunti. Durante un'operazione di esportazione, determina il percorso in cui salvare il file di configurazione dell'installazione. Deve essere un percorso assoluto.
--help, --?, -h, -? Visualizzare l'utilizzo dei parametri.


opzione del layout della riga di comando Descrizione
--layout <dir> Specifica una directory per creare una cache di installazione offline.
--lang <one or more language-locales> Usato con --layout per preparare una cache di installazione offline con pacchetti di risorse con le lingue specificate. Vedere un elenco completo delle impostazioni locali della lingua supportate di seguito.
--add <one or more workload or component IDs> Uno o più ID del carico di lavoro o del componente da aggiungere. I componenti necessari dell'artefatto vengono installati, ma non i componenti consigliati o facoltativi. È possibile controllare componenti aggiuntivi a livello globale usando --includeRecommended e/o --includeOptional. Per un controllo con granularità più fine, è possibile aggiungere ;includeRecommended o ;includeOptional al ID, (ad esempio --add Workload1;includeRecommended o --add Workload2;includeOptional).
Nota: se viene usato --add vengono scaricati solo i carichi di lavoro e i componenti specificati e le relative dipendenze. Se --add non è specificato, tutti i carichi di lavoro e i componenti vengono scaricati nel layout.
--includeRecommended Include i componenti consigliati per tutti i carichi di lavoro installati, ma non i componenti facoltativi. I carichi di lavoro vengono specificati con --allWorkloads o --add.
--includeOptional Include i componenti facoltativi and consigliati per i carichi di lavoro inclusi nel layout. I carichi di lavoro vengono specificati con --add.
--keepLayoutVersion Applica le modifiche al layout senza aggiornare la versione del layout.
--verify Verificare il contenuto di un layout. Sono elencati tutti i file danneggiati o mancanti.
--fix Verificare il contenuto di un layout. Se sono presenti file danneggiati o mancanti, vengono scaricati di nuovo. l'accesso a Internet è necessario per correggere un layout.
--clean <one or more paths to catalogs> Rimuove le versioni precedenti dei componenti da un layout aggiornato a una versione più recente.


Opzione della riga di comando avanzata Descrizione
--channelId <id> ID del canale per l'istanza da installare. È necessario per il comando di installazione, ignorato per altri comandi se è specificato --installPath.
--channelUri <uri> URI del manifesto del canale. Se gli aggiornamenti non sono desiderati, --channelUri può puntare a un file inesistente. (ad esempio-- channelUri C:\doesntExist.chman) Può essere usato per il comando di installazione; viene ignorato per altri comandi.
--installChannelUri <uri> URI del manifesto del canale da utilizzare per l'installazione. L'URI specificato da --channelUri (che deve essere specificato quando viene specificato --installChannelUri) viene usato per rilevare gli aggiornamenti. Può essere utilizzato per il comando di installazione; viene ignorato per altri comandi.
--installCatalogUri <uri> URI del manifesto del catalogo da utilizzare per l'installazione. Se specificato, il manager del canale tenta di scaricare il manifesto del catalogo da questo URI prima di usare l'URI nel manifesto del canale di installazione. Questo parametro viene utilizzato per supportare l'installazione offline, in cui verrà creata la cache del layout con il catalogo prodotti già scaricato. Può essere usato per il comando di installazione; viene ignorato per altri comandi.
--productId <id> ID del prodotto per l'istanza che verrà installata. Questa operazione viene prepopolata nelle normali condizioni di installazione.
--newChannelUri <uri> Nuovo URI o percorso del manifesto del canale da usare per gli aggiornamenti.
--layoutUri <uri> URI del layout ospitato sul Web usato per recuperare i pacchetti di installazione.
--theme <theme> Modifica il tema dei colori del programma di installazione. I valori possono essere Chiaro/Scuro/Contrasto elevato.
--wait Il processo attenderà il completamento dell'installazione prima di restituire un codice di uscita. Ciò è utile quando si automatizzano le installazioni in cui è necessario attendere il completamento dell'installazione per gestire il codice restituito da tale installazione.
--locale <language-locale> Modificare la lingua di visualizzazione dell'interfaccia utente per il programma di installazione stesso. L'impostazione verrà mantenuta. Vedere un elenco completo delle impostazioni locali della lingua supportate di seguito.
--cache Se presente, i pacchetti verranno mantenuti dopo l'installazione per le riparazioni successive. Questa impostazione sostituisce l'impostazione dei criteri globali da utilizzare per installazioni, riparazioni o modifiche successive. Il criterio predefinito prevede la memorizzazione nella cache dei pacchetti. Questa operazione viene ignorata per il comando di disinstallazione.
--nocache Se presente, i pacchetti verranno eliminati dopo l'installazione o il ripristino. Verranno scaricati di nuovo solo se necessario ed eliminati di nuovo dopo l'uso. Questa impostazione sostituisce l'impostazione dei criteri globali da utilizzare per installazioni, riparazioni o modifiche successive. I criteri predefiniti consentono di memorizzare nella cache i pacchetti.
--noUpdateInstaller Se presente, impedisce al programma di installazione di aggiornarsi quando viene specificato non interattiva. Il programma di installazione non riuscirà a eseguire il comando e restituirà un codice di uscita diverso da zero se noUpdateInstaller viene specificato con non interattiva quando è necessario un aggiornamento del programma di installazione.
--noWeb Se presente, il programma di installazione di Visual Studio usa i file nella directory di layout per installare Visual Studio. Se un utente tenta di installare componenti non inclusi nel layout, l'installazione non riesce.
--path <name>=<path> Utilizzato per specificare percorsi di installazione personalizzati per l'installazione. I nomi di percorso supportati sono condivisi, memorizzati nella cache e installati.
--path cache=<path> Utilizza il percorso specificato per scaricare i file di installazione. Questo percorso può essere impostato solo la prima volta che viene installato Visual Studio. Esempio: --path cache="C:\VS\cache"
--path shared=<path> Contiene file condivisi per installazioni side-by-side Visual Studio. Alcuni strumenti e SDK vengono installati in un percorso in questa unità, mentre altri potrebbero eseguire l'override di questa impostazione e installare in un'altra unità. esempio: --path shared="C:\VS\shared"

Important: può essere impostato una sola volta e alla prima installazione di Visual Studio.
--path install=<path> Equivalente a –-installPath. In particolare, --installPath "C:\VS" e --path install="C:\VS" sono equivalenti. è possibile usare solo uno di questi elementi alla volta.
