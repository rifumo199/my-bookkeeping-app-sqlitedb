Install command-line option 설명
--installPath <dir> 인스턴스가 작동할 설치 디렉터리입니다. 설치 명령의 경우 선택 사항이며 인스턴스를 설치할 위치입니다. 다른 명령의 경우 이 명령이 필요하며 이전에 설치한 인스턴스가 설치된 위치입니다.
--addProductLang <language-locale> 설치 또는 수정 작업 중에 제품에 설치된 UI 언어 팩을 결정합니다. 명령줄에 여러 언어 팩을 추가하기 위해 여러 번 나타날 수 있습니다. 없는 경우 설치에서 컴퓨터 로캘을 사용합니다. 아래에서 지원되는 언어 로캘의 전체 목록을 확인하세요.
--removeProductLang <language-locale> 설치 또는 수정 작업 중 제품에서 제거할 UI 언어 팩을 결정합니다. 명령줄에 여러 언어 팩을 추가하기 위해 여러 번 나타날 수 있습니다. 아래에서 지원되는 언어 로캘의 전체 목록을 확인하세요.
--add <one or more workload or component IDs> 추가할 하나 이상의 워크로드 또는 구성 요소 ID입니다. 아티팩트에 필요한 구성 요소가 설치되어 있지만 권장 구성 요소나 선택적 구성 요소는 설치되지 않았습니다. --includeRecommended 및/또는 --includeOptional를 사용하여 전역으로 추가 구성 요소를 제어할 수 있습니다. 여러 워크로드 또는 구성 요소를 포함하려면 --add 명령(예: --add Workload1 --add Workload2)을 반복합니다. 세분화된 컨트롤의 경우 ID에 ;includeRecommended 또는 ;includeOptional을 추가할 수 있습니다. (예: --add Workload1;includeRecommended 또는 --add Workload2;includeRecommended;includeOptional). 필요에 따라 이 옵션을 반복할 수 있습니다.
--remove <one or more workload or component IDs> 제거할 하나 이상의 워크로드 또는 구성 요소 ID입니다.
--in <path> 응답 파일의 URI 또는 ​​경로입니다.
--all 제품에 대한 모든 워크로드 및 구성 요소를 설치할지 여부입니다.
--allWorkloads 모든 워크로드와 구성 요소를 설치합니다. 권장 구성 요소나 선택적 구성 요소는 설치하지 않습니다.
--includeRecommended 설치된 워크로드에 대한 권장 구성 요소를 포함하지만 선택적 구성 요소는 포함하지 않습니다. 워크로드가 --allWorkloads 또는 --add를 사용하여 지정됩니다.
--includeOptional 설치된 워크로드에 대한 선택적 구성 요소를 포함하지만 권장 구성 요소는 포함하지 않습니다. 워크로드가 --allWorkloads 또는 --add를 사용하여 지정됩니다.
--quiet, -q 설치를 수행하는 동안 사용자 인터페이스를 표시하지 마세요.
--passive, -p 사용자 인터페이스를 표시하지만 사용자에게 상호 작용을 요청하지 않습니다.
--norestart 있는 경우, --passive 또는 --quiet의 명령은 컴퓨터를 자동으로 다시 시작하지 않습니다(필요한 경우). --passive 및 --quiet가 모두 지정되지 않은 경우 무시됩니다.
--nickname <name> 설치된 제품에 할당할 애칭을 정의합니다. 애칭은 10자를 초과할 수 없습니다.
--productKey <key> 설치된 제품에 사용할 제품 키를 정의합니다. xxxxx-xxxxx-xxxxx-xxxxx-xxxxx 또는 xxxxxxxxxxxxxxxxxxxxxxxxx 형식의 25자의 영숫자로 구성됩니다.
--config <fullpath> 설치 또는 수정 작업 중에는 추가할 워크로드 및 구성 요소가 결정됩니다. 이전에 저장한 설치 구성 파일을 기반으로 합니다. 이 작업은 추가 작업이며 워크로드 또는 구성 요소를 제거하지 않습니다. 파일에 없는 경우 또한 제품에 적용되지 않는 항목은 추가되지 않습니다. 내보내기 작업 동안 설치 구성 파일을 저장할 위치를 결정합니다. 절대 경로여야 합니다.
--help, --?, -h, -? 매개 변수 사용량을 표시합니다.


레이아웃 명령줄 옵션 설명
--layout <dir> 오프라인 설치 캐시를 생성할 디렉터리를 지정합니다.
--lang <one or more language-locales> --layout과 사용되여 지정된 언어의 리소스 패키지와 함께 오프라인 설치 캐시를 준비합니다. 아래에서 지원되는 언어 로캘의 전체 목록을 확인하세요.
--add <one or more workload or component IDs> 추가할 하나 이상의 워크로드 또는 구성 요소 ID입니다. 아티팩트에 필요한 구성 요소가 설치되어 있지만 권장 구성 요소나 선택적 구성 요소는 설치되지 않았습니다. --includeRecommended 및/또는 --includeOptional을 사용하여 전역적으로 추가 구성 요소를 제어할 수 있습니다. 세분화된 컨트롤의 경우 ;includeRecommended 또는 ;includeOptional ID에 추가할 수 있습니다. (예: --add Workload1;includeRecommended 또는 --add Workload2;includeOptional).
참고: --add를 사용하는 경우 지정된 워크로드 및 구성 요소 및 해당 종속성만 다운로드됩니다. --add를 지정하지 않으면 모든 워크로드와 구성 요소가 레이아웃에 다운로드됩니다.
--includeRecommended 설치된 워크로드에 대한 권장 구성 요소를 포함하지만 선택적 구성 요소는 포함하지 않습니다. 워크로드가 --allWorkloads 또는 --add를 사용하여 지정됩니다.
--includeOptional 레이아웃에 포함되는 워크로드에 권장되는 구성 요소 및 선택적 구성 요소를 포함합니다. 워크로드는 --add를 사용하여 지정됩니다.
--keepLayoutVersion 레이아웃 버전을 업데이트하지 않고 레이아웃에 변경 내용을 적용합니다.
--verify 레이아웃의 내용을 확인합니다. 손상되거나 누락된 파일이 나열됩니다.
--fix 레이아웃의 내용을 확인합니다. 파일이 손상되었거나 누락된 경우 다시 다운로드됩니다. 레이아웃을 수정하려면 인터넷에 액세스해야 합니다.
--clean <one or more paths to catalogs> 최신 버전으로 업데이트된 레이아웃에서 이전 버전의 구성 요소를 제거합니다.


고급 명령줄 옵션 설명
--channelId <id> 인스턴스를 설치할 채널의 ID입니다. --installPath를 지정한 경우 다른 명령에 대해 무시되는 설치 명령에 필요합니다.
--channelUri <uri> 채널 매니페스트의 URI입니다. 업데이트가 필요한 경우 --channelUri가 존재하지 않는 파일을 가리킬 수 있습니다. (예: --channelUri C:\doesntExist.chman) 설치 명령에 사용할 수 있습니다. 다른 명령에 대해 무시됩니다.
--installChannelUri <uri> 설치에 사용할 채널 매니페스트의 URI입니다. --channelUri로 지정된 URI는 (--installChannelUri가 지정될 때 지정되어야 함)는 업데이트를 감지하는 데 사용됩니다. 설치 명령에 사용할 수 있습니다. 다른 명령에 대해 무시됩니다.
--installCatalogUri <uri> 설치에 사용할 카탈로그 매니페스트의 URI입니다. 지정하면 채널 관리자가 설치 채널 매니페스트의 URI를 사용하기 전에 이 URI에서 카탈로그 매니페스트를 다운로드하려고 시도합니다. 이 매개 변수는 이미 다운로드한 제품 카탈로그를 사용하여 레이아웃 캐시가 만들어지는 오프라인 설치를 지원하는 데 사용됩니다. 설치 명령에 사용할 수 있습니다. 다른 명령에 대해 무시됩니다.
--productId <id> 설치할 인스턴스의 제품 ID입니다. 일반 설치 조건에서 미리 채워진 것입니다.
--newChannelUri <uri> 업데이트에 사용할 채널 매니페스트의 새 URI 또는 ​​경로입니다.
--layoutUri <uri> 설치 패키지를 가져오는 데 사용되는 웹 호스팅 레이아웃 URI입니다.
--theme <theme> 설치 프로그램의 색상 테마를 변경합니다. 값은 Light/Dark/HighContrast가 될 수 있습니다.
--wait 설치가 완료될 때까지 기다린 후 종료 코드를 반환합니다. 설치가 완료되기를 기다려야 해당 설치의 반환 코드를 처리할 때까지 기다려야 하는 설치를 자동화할 때 유용합니다.
--locale <language-locale> 설치 관리자 자체에 대한 사용자 인터페이스의 표시 언어를 변경합니다. 설정이 유지됩니다. 아래에서 지원되는 언어 로캘의 전체 목록을 확인하세요.
--cache 패키지가 있는 경우 이후 복구를 위해 설치 후 패키지가 유지됩니다. 이 설정은 이후 설치, 복구 또는 수정에 사용할 전역 정책 설정을 재정의합니다. 기본 정책은 패키지를 캐시하는 것입니다. 제거 명령에 대해 무시됩니다.
--nocache 있는 경우 설치 또는 복구 후 패키지가 삭제됩니다. 필요한 경우에만 다시 다운로드되고 사용 후에 다시 삭제됩니다. 이 설정은 이후 설치, 복구 또는 수정에 사용할 전역 정책 설정을 재정의합니다. 기본 정책은 패키지를 캐시하는 것입니다.
--noUpdateInstaller 있는 경우 quiet가 지정된 경우 설치 관리자가 자신을 업데이트하지 못하도록 합니다. 설치 관리자 업데이트가 필요할 때 noUpdateInstaller가 quiet로 지정된 경우 설치 관리자가 명령을 실패하고 0이 아닌 종료 코드를 반환합니다.
--noWeb 있는 경우 Visual Studio 설치 프로그램에서 레이아웃 디렉터리의 파일을 사용하여 Visual Studio를 설치합니다. 사용자가 레이아웃에 없는 구성 요소를 설치하려고 하면 설치가 실패합니다.
--path <name>=<path> 설치를 위한 사용자 지정 설치 경로를 지정하는 데 사용됩니다. 지원되는 경로 이름은 공유, 캐시 및 설치입니다.
--path cache=<path> 지정한 위치를 사용하여 설치 파일을 다운로드합니다. 이 위치는 Visual Studio 처음 설치된 경우에만 설정할 수 있습니다. 예: --path cache="C:\VS\cache"
--path shared=<path> side-by-side Visual Studio 설치를 위한 공유 파일이 포함되어 있습니다. 일부 도구 및 SDK는 이 드라이브의 위치에 설치되지만 다른 도구와 SDK는 이 설정을 무시하고 다른 드라이브에 설치할 수 있습니다. 예: --path shared="C:\VS\shared"

중요: Visual Studio에 처음 설치된 경우에만 설정할 수 있습니다.
--path install=<path> –installPath와 같습니다. 특히 --installPath "C:\VS" 및 --path install="C:\VS"와 동일합니다. 한 번에 하나만 사용할 수 있습니다.
