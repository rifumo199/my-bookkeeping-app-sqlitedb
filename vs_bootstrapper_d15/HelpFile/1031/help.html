Microsoft Visual Studio 2022

Visual Studio-Befehlszeile

Syntax: vs_enterprise.exe [Befehl [Optionen]]

(Ersetzen Sie vs_enterprise.exe ggf. für die zu installierende Produktversion.)

Befehlszeilenbefehl Beschreibung
(blank) Installiert das Produkt.
modify Ändert ein installiertes Produkt.
update Aktualisiert ein installiertes Produkt.
repair Repariert ein installiertes Produkt.
resume Setzt die Installation fort.
export Exportiert eine Konfigurationsdatei für die Installation.
modifySettings Ändert die Einstellungen einer vorhandenen Installation.
removeChannel Entfernt einen Kanal.
updateAll Aktualisiert alle anwendbaren Installationen.
uninstall Deinstalliert ein installiertes Produkt.


Install command-line option Beschreibung
--installPath <dir> Das Installationsverzeichnis, für das die Instanz eine Aktion ausführen soll. Für den Installationsbefehl ist dies optional und der Ort, an dem die Instanz installiert wird. Für andere Befehle ist dies erforderlich und der Ort, an dem die zuvor installierte Instanz installiert wurde.
--addProductLang <language-locale> Während eines Installations- oder Änderungsvorgangs bestimmt dies die Benutzeroberflächensprachpakete, die im Produkt installiert werden sollen. Es kann mehrmals in der Befehlszeile angezeigt werden, um mehrere Sprachpakete hinzuzufügen. Falls nicht vorhanden, wird bei der Installation das Gebietsschema des Computers verwendet. Eine vollständige Liste der unterstützten Sprachgebietsschemas finden Sie unten.
--removeProductLang <language-locale> Während eines Installations- oder Änderungsvorgangs bestimmt dies die Benutzeroberflächensprachpakete, die aus dem Produkt entfernt werden sollen. Es kann mehrmals in der Befehlszeile angezeigt werden, um mehrere Sprachpakete hinzuzufügen. Eine vollständige Liste der unterstützten Sprachgebietsschemas finden Sie unten.
--add <one or more workload or component IDs> Mindestens eine Workload- oder Komponenten-ID, die hinzugefügt werden soll. Die erforderlichen Komponenten des Artefakts werden installiert, jedoch nicht die empfohlenen oder optionalen Komponenten. Mit --includeRecommended und/oder --includeOptional können Sie zusätzliche Komponenten global steuern. Um mehrere Workloads oder Komponenten einzuschließen, wiederholen Sie den Befehl --add (z. B. --add Workload1 --add Workload2). Für eine differenziertere Steuerung können Sie ;includeRecommended oder ;includeOptional an die ID anfügen. (Beispiel: --add Workload1;includeRecommended oder --add Workload2;includeRecommended;includeOptional). Sie können diese Option bei Bedarf wiederholen.
--remove <one or more workload or component IDs> Mindestens eine Workload- oder Komponenten-ID, die entfernt werden soll.
--in <path> Der URI oder Pfad zu einer Antwortdatei.
--all Gibt an, ob alle Workloads und Komponenten für ein Produkt installiert werden sollen.
--allWorkloads Installiert alle Workloads und Komponenten, aber keine empfohlenen oder optionalen Komponenten.
--includeRecommended Enthält die empfohlenen Komponenten für alle installierten Workloads, aber nicht die optionalen Komponenten. Die Workloads werden entweder mit --allWorkloads oder mit --add angegeben.
--includeOptional Enthält die optionalen Komponenten für alle installierten Workloads, aber nicht die empfohlenen Komponenten. Die Workloads werden entweder mit --allWorkloads oder mit --add angegeben.
--quiet, -q Während der Installation keine Benutzeroberfläche anzeigen.
--passive, -p Die Benutzeroberfläche anzeigen, jedoch keine Interaktion vom Benutzer anfordern.
--norestart Falls vorhanden, starten Befehle mit --passive oder --quiet den Computer nicht automatisch neu (falls erforderlich). Dies wird ignoriert, wenn weder --passive noch --quiet angegeben ist.
--nickname <name> Dies definiert den Spitznamen, der einem installierten Produkt zugewiesen werden soll. Der Spitzname darf nicht länger als 10 Zeichen sein.
--productKey <key> Definiert den Product Key, der für ein installiertes Produkt verwendet werden soll. Er besteht aus 25 alphanumerischen Zeichen im Format xxxxx-xxxxx-xxxxx-xxxxx-xxxxx or xxxxxxxxxxxxxxxxxxxxxxxxx.
--config <fullpath> Während eines Installations- oder Änderungsvorgangs bestimmt dies die Workloads und Komponenten, die hinzugefügt werden sollen, basierend auf einer zuvor gespeicherten Installationskonfigurationsdatei. Dieser Vorgang ist additiv, und es werden keine Workloads oder Komponenten entfernt, wenn sie nicht in der Datei vorhanden sind. Außerdem werden keine Elemente hinzugefügt, die nicht für das Produkt gelten. Während eines Exportvorgangs bestimmt dies den Speicherort für die Installationskonfigurationsdatei. Dies muss ein absoluter Pfad sein.
--help, --?, -h, -? Parameterverwendung anzeigen.


Layout-Befehlszeilenoption Beschreibung
--layout <dir> Gibt ein Verzeichnis zum Erstellen eines Offlineinstallationscaches an.
--lang <one or more language-locales> Wird mit --layout verwendet, um einen Offlineinstallationscache mit Ressourcenpaketen mit den angegebenen Sprachen vorzubereiten. Eine vollständige Liste der unterstützten Sprachgebietsschemas finden Sie unten.
--add <one or more workload or component IDs> Mindestens eine Workload- oder Komponenten-ID, die hinzugefügt werden soll. Die erforderlichen Komponenten des Artefakts werden installiert, jedoch nicht die empfohlenen oder optionalen Komponenten. Mit --includeRecommended und/oder --includeOptional können Sie zusätzliche Komponenten global steuern. Für eine differenziertere Steuerung können Sie ;includeRecommended oder ;includeOptional an die ID anfügen. (Beispiel: --add Workload1;includeRecommended oder --add Workload2;includeOptional).
Hinweis: Wenn --add verwendet wird, werden nur die angegebenen Workloads und Komponenten und deren Abhängigkeiten heruntergeladen. Wenn --add nicht angegeben ist, werden alle Workloads und Komponenten in das Layout heruntergeladen.
--includeRecommended Enthält die empfohlenen Komponenten für alle installierten Workloads, aber nicht die optionalen Komponenten. Die Workloads werden entweder mit --allWorkloads oder --add angegeben.
--includeOptional Enthält die empfohlenen und optionalen Komponenten für alle Workloads, die in das Layout eingeschlossen werden. Die Workloads werden mit --add angegeben.
--keepLayoutVersion Änderungen auf das Layout anwenden, ohne die Version des Layouts zu aktualisieren.
--verify Den Inhalt eines Layouts überprüfen. Alle beschädigten oder fehlenden Dateien werden aufgelistet.
--fix Den Inhalt eines Layouts überprüfen. Wenn Dateien beschädigt sind oder fehlen, werden sie erneut heruntergeladen. Zum Reparieren eines Layouts ist Internetzugriff erforderlich.
--clean <one or more paths to catalogs> Entfernt alte Versionen von Komponenten aus einem Layout, das auf eine neuere Version aktualisiert wurde.


Erweiterte Befehlszeilenoption Beschreibung
--channelId <id> Die ID des Kanals für die Instanz, die installiert werden soll. Dies ist für den Installationsbefehl erforderlich und wird für andere Befehle ignoriert, wenn --installPath angegeben ist.
--channelUri <uri> Der URI des Kanalmanifests. Wenn keine Updates gewünscht werden, kann --channelUri auf eine nicht vorhandene Datei verweisen. (Beispiel: --channelUri C:\doesntExist.chman) Dies kann für den Installationsbefehl verwendet werden und wird für andere Befehle ignoriert.
--installChannelUri <uri> Der URI des Kanalmanifests, das für die Installation verwendet werden soll. Der durch--channelUri angegebene URI (der angegeben werden muss, wenn --installChannelUri angegeben ist) wird zum Erkennen von Updates verwendet. Dies kann für den Installationsbefehl verwendet werden und wird für andere Befehle ignoriert.
--installCatalogUri <uri> Der URI des Katalogmanifests, das für die Installation verwendet werden soll. Ist der URI angegeben, versucht der Kanal-Manager, das Katalogmanifest von diesem URI herunterzuladen, bevor der URI im Installationskanalmanifest verwendet wird. Dieser Parameter wird zur Unterstützung der Offlineinstallation verwendet, bei der der Layoutcache mit dem bereits heruntergeladenen Produktkatalog erstellt wird. Dies kann für den Installationsbefehl verwendet werden und wird für andere Befehle ignoriert.
--productId <id> Die ID des Produkts für die Instanz, die installiert wird. Diese ist unter normalen Installationsbedingungen aufgefüllt.
--newChannelUri <uri> Der neue URI oder Pfad zum Kanalmanifest, das für Updates verwendet werden soll.
--layoutUri <uri> Der webgehostete Layout-URI, der zum Abrufen von Installationspaketen verwendet wird.
--theme <theme> Ändert das Farbdesign des Installers. Die Werte können Hell/Dunkel/HighContrast sein.
--wait Der Prozess wartet, bis die Installation abgeschlossen ist, bevor ein Exitcode zurückgegeben wird. Dies ist nützlich, wenn Installationen automatisiert werden, bei denen auf den Abschluss der Installation gewartet werden muss, um den Rückgabecode der jeweiligen Installation zu verarbeiten.
--locale <language-locale> Die Anzeigesprache der Benutzeroberfläche für den Installer selbst ändern. Die Einstellung wird beibehalten. Eine vollständige Liste der unterstützten Sprachgebietsschemas finden Sie unten.
--cache Falls vorhanden, werden Pakete nach der Installation für nachfolgende Reparaturen beibehalten. Hiermit wird die globale Richtlinieneinstellung außer Kraft gesetzt, die für nachfolgende Installationen, Reparaturen oder Änderungen verwendet werden soll. Die Standardrichtlinie besteht im Zwischenspeichern von Paketen. Dies wird für den Deinstallationsbefehl ignoriert.
--nocache Falls vorhanden, werden Pakete nach der Installation oder Reparatur gelöscht. Sie werden nur bei Bedarf erneut heruntergeladen und nach der Verwendung erneut gelöscht. Hiermit wird die globale Richtlinieneinstellung außer Kraft gesetzt, die für nachfolgende Installationen, Reparaturen oder Änderungen verwendet werden soll. Die Standardrichtlinie besteht im Zwischenspeichern von Paketen.
--noUpdateInstaller Falls vorhanden, wird verhindert, dass der Installer sich selbst aktualisiert, wenn "quiet" angegeben ist. Der Installer führt den Befehl nicht aus, und es wird ein Exitcode ungleich Null zurückgegeben, wenn noUpdateInstaller mit "quiet" angegeben ist, sofern ein Installerupdate erforderlich ist.
--noWeb Falls vorhanden, verwendet Visual Studio Setup die Dateien in Ihrem Layoutverzeichnis, um Visual Studio zu installieren. Wenn ein Benutzer versucht, Komponenten zu installieren, die sich nicht im Layout befinden, schlägt das Setup fehl.
--path <name>=<path> Wird verwendet, um benutzerdefinierte Installationspfade für die Installation anzugeben. Unterstützte Pfadnamen sind "shared", "cache" und "install".
--path cache=<path> Verwendet den von Ihnen angegebenen Speicherort zum Herunterladen von Installationsdateien. Dieser Speicherort kann nur festgelegt werden, wenn Visual Studio zum ersten Mal installiert wird. Beispiel: --path cache="C:\VS\cache"
--path shared=<path> Enthält freigegebene Dateien für parallele Visual Studio-Installationen. Einige Tools und SDKs werden an einem Speicherort auf diesem Laufwerk installiert, während andere diese Einstellung möglicherweise außer Kraft setzen und auf einem anderen Laufwerk installieren. Beispiel: --path shared="C:\VS\shared"

Wichtig: Dies kann nur einmal festgelegt werden, und zwar beim ersten Installieren Visual Studio.
--path install=<path> Entspricht –-installPath. Insbesondere --installPath "C:\VS" und --path install="C:\VS" sind äquivalent. Es kann jeweils nur einer dieser Befehle verwendet werden.
