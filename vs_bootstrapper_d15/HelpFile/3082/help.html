Microsoft Visual Studio 2022

línea de comandos de Visual Studio

Uso: vs_enterprise.exe [comando [opciones]]

(Reemplace vs_enterprise.exe según corresponda para la edición del producto que va a instalar).

Comando de línea de comandos Descripción
(blank) Instala el producto.
modify Modifica un producto instalado.
update Actualiza un producto instalado.
repair Repara un producto instalado.
resume Reanudar la instalación.
export Exporta un archivo de configuración de la instalación.
modifySettings Modifica la configuración de una instalación existente.
removeChannel Quita un canal.
updateAll Actualiza todas las instalaciones aplicables.
uninstall Desinstala un producto instalado.


instalar la opción de la línea de comandos Descripción
--installPath <dir> El directorio de instalación en el que actuará la instancia. Para el comando de instalación, es opcional y es donde se instalará la instancia. Para los otros comandos, es necesario y es donde se instaló la instancia instalada previamente.
--addProductLang <language-locale> Durante una operación de instalación o modificación, esto determina los paquetes de idioma de la interfaz de usuario que están instalados en el producto. Puede aparecer varias veces en la línea de comandos para agregar varios paquetes de idioma. Si no está presente, la instalación usa la configuración regional del equipo. Consulte a continuación una lista completa de configuraciones regionales de idioma admitidas.
--removeProductLang <language-locale> Durante una operación de instalación o modificación, esto determina los paquetes de idioma de la interfaz de usuario que se van a quitar del producto. Puede aparecer varias veces en la línea de comandos para agregar varios paquetes de idioma. Consulte a continuación una lista completa de configuraciones regionales de idioma admitidas.
--add <uno o varios identificadores de componente o carga de trabajo> Uno o más identificadores de componente o carga de trabajo que se van a agregar. Los componentes necesarios del artefacto están instalados, pero no los componentes recomendados u opcionales. Puede controlar componentes adicionales globalmente mediante --includeRecommended o --includeOptional. Para incluir varias cargas de trabajo o componentes, repita el comando --add (por ejemplo, --add Workload1 --add Workload2). Para un control más preciso, puede anexar ;includeRecommended o ;includeOptional al identificador (por ejemplo, --add Workload1;includeRecommended o --add Workload2;includeRecommended;includeOptional). Puede repetir esta opción según sea necesario.
--remove <uno o varios identificadores de componente o carga de trabajo> Uno o varios id. de componente o carga de trabajo que se van a quitar.
--in <path> El URI o la ruta de acceso a un archivo de respuesta.
--all Si se van a instalar todas las cargas de trabajo y componentes de un producto.
--allWorkloads Instala todas las cargas de trabajo y componentes, los componentes no recomendados u opcionales.
--includeRecommended Incluye los componentes recomendados para todas las cargas de trabajo que están instaladas, pero no los componentes opcionales. Las cargas de trabajo se especifican con --allWorkloads o --add.
--includeOptional Incluye los componentes opcionales para todas las cargas de trabajo que están instaladas, pero no los componentes recomendados. Las cargas de trabajo se especifican con --allWorkloads o --add.
--quiet, -q No muestre ninguna interfaz de usuario al realizar la instalación.
--passive, -p Muestra la interfaz de usuario, pero no solicita ninguna interacción del usuario.
--norestart Si están presentes, los comandos con --passive o --quiet no reiniciarán automáticamente la máquina (si es necesario). Se omite si no se especifica --passive ni --quiet.
--nickname <name> Esto define el alias que se va a asignar a un producto instalado. El alias no puede tener más de 10 caracteres.
--productKey <key> Esto define la clave de producto que se va a usar para un producto instalado. Se compone de 25 caracteres alfanuméricos con el formato xxxxx-xxxxx-xxxxx-xxxxx-xxxxx o xxxxxxxxxxxxxxxxxxxxxxxxx.
--config <fullpath> Durante una operación de instalación o modificación, esto determina las cargas de trabajo y los componentes que se van a agregar basado en un archivo de configuración de instalación guardado previamente. Esta operación es aditiva y no quitará ninguna carga de trabajo o componente si no están presentes en el archivo. Además, no se agregarán los elementos que no se aplican al producto. Durante una operación de exportación, determina la ubicación para guardar el archivo de configuración de instalación. Debe ser una ruta de acceso absoluta.
--help, --?, -h, -? Muestra el uso de los parámetros.


opción de la línea de comandos de Diseño Descripción
--layout <dir> Especifica un directorio para crear una caché de instalación sin conexión.
--lang <uno o varios lenguajes regionales> Se usa con --layout para preparar una memoria caché de instalación sin conexión con paquetes de recursos con los idiomas especificados. Vea una lista completa de las configuraciones regionales de idioma admitidas a continuación.
--add <uno o varios identificadores de componente o carga de trabajo> Uno o más identificadores de componente o carga de trabajo que se van a agregar. Los componentes necesarios del artefacto están instalados, pero no los componentes recomendados u opcionales. Puede controlar componentes adicionales globalmente mediante --includeRecommended o --includeOptional. Para un control más preciso, puede anexar ;includeRecommended o ;includeOptional al identificador (por ejemplo, --add Workload1;includeRecommended o --add Workload2;includeOptional).
Nota: si se usa --add, solo se descargan las cargas de trabajo y los componentes especificados y sus dependencias. Si no se especifica --add, todas las cargas de trabajo y componentes se descargan en el diseño.
--includeRecommended Incluye los componentes recomendados para todas las cargas de trabajo que están instaladas, pero no los componentes opcionales. Las cargas de trabajo se especifican con --allWorkloads o --add.
--includeOptional Incluye los componentes recomendados and opcionales para las cargas de trabajo que se incluyen en el diseño. Las cargas de trabajo se especifican con --add.
--keepLayoutVersion Aplicar los cambios al diseño sin actualizar la versión del diseño.
--verify Compruebe el contenido de un diseño. Todos los archivos dañados o que faltan aparecen en la lista.
--fix Compruebe el contenido de un diseño. Si se detecta que algunos archivos están dañados o faltan, se vuelven a descargar. Se requiere acceso a Internet para corregir un diseño.
--clean <una o varias rutas de acceso a los catálogos> Quita las versiones anteriores de los componentes desde un diseño que se ha actualizado a una versión más reciente.


Opción de línea de comandos avanzada Descripción
--channelId <id> El Identificador del canal para la instancia que se va a instalar. Esto es necesario para el comando de instalación, se omite para los otros comandos si --installPath se especifica.
--channelUri <uri> El URI del manifiesto del canal. Si no se desean actualizaciones, : channelUri puede apuntar a un archivo que no existe. (por ejemplo, --channelUri C:\doesntExist.chman) Se puede usar para el comando de instalación; se omite para los otros comandos.
--installChannelUri <uri> URI del manifiesto del canal que se va a usar para la instalación. URI especificado por --channelUri (que debe especificarse cuando --installChannelUri se especifica) para detectar las actualizaciones. Se puede usar para el comando de instalación; se omite para los otros comandos.
--installCatalogUri <uri> URI del manifiesto del catálogo que se va a usar para la instalación. Si se especifica, el administrador de canales intenta descargar el manifiesto del catálogo desde este URI antes de usar el URI en el manifiesto del canal de instalación. Este parámetro se usa para admitir la instalación sin conexión, donde la memoria caché de diseño se creará con el catálogo de productos ya descargado. Se puede usar para el comando de instalación; se omite para los otros comandos.
--productId <id> El Identificador del producto para la instancia que se instalará. Esto se rellena previamente en condiciones de instalación normales.
--newChannelUri <uri> El nuevo URI o ruta de acceso al manifiesto del canal que se va a usar para las actualizaciones.
--layoutUri <uri> El URI del diseño alojado en la web que se ha utilizado para obtener los paquetes de instalación.
--theme <theme> Cambia el tema de color del instalador. Los valores pueden ser Claro/Oscuro/AltoContraste.
--wait El proceso esperará hasta que se complete la instalación antes de devolver un código de salida. Esto es útil cuando se automatizan las instalaciones en las que hay que esperar a que finalice la instalación para controlar el código revuelto desde esa instalación.
--locale <language-locale> Cambie el idioma del diseño de la interfaz de usuario para el propio instalador. La configuración se conservará. Consulte a continuación una lista completa de configuraciones regionales de idioma admitidas.
--cache Si está presente, los paquetes se conservarán después de instalarse para las reparaciones posteriores. Esto invalida la configuración de directiva global que se usará para las siguientes instalaciones, reparaciones o modificaciones. La directiva predeterminada es almacenar paquetes en caché. Esto se omite para el comando de desinstalación.
--nocache Si está presente, los paquetes se eliminarán después de instalarse o repararse. Solo se descargarán de nuevo si es necesario y se eliminarán de nuevo después de su uso. Esto invalida la configuración de directiva global que se usará para las siguientes instalaciones, reparaciones o modificaciones. La directiva predeterminada es almacenar paquetes en caché.
--noUpdateInstaller Si está presente, impide que el instalador se actualice automáticamente cuando se especifica un modo silencioso. El instalador producirá un fallo en el comando y devolverá un código de salida distinto de cero si noUpdateInstaller se especifica sin notificaciones cuando se requiere una actualización del instalador.
--noWeb Si está presente, la configuración de Visual Studio usa los archivos del directorio de diseño para instalar Visual Studio. Si un usuario intenta instalar componentes que no están en el diseño, se produce un fallo en el programa de instalación.
--path <name>=<path> Se usa para especificar las rutas de instalación personalizadas. Los nombres de la ruta admitidos se comparten, almacenan en caché e instalan.
--path cache=<path> Usa la ubicación que especifique para descargar los archivos de instalación. Esta ubicación solo se puede establecer la primera vez que se instala Visual Studio. Ejemplo: --path cache="C:\VS\cache"
--path shared=<path> Contiene archivos compartidos para instalaciones de Visual Studio en paralelo. Algunas herramientas y SDK se instalan en una ubicación de esta unidad, mientras que otros podrían invalidar esta configuración e instalar en otra unidad. Ejemplo: --path shared="C:\VS\shared"

Important: Solo se puede establecer una vez y la primera vez que se instala Visual Studio.
--path install=<path> Equivalente a –-installPath. En concreto, --installPath "C:\VS" y --path install="C:\VS" son equivalentes. Solo se puede usar uno de ellos a la vez.
