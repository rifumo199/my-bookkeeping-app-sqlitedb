Microsoft Visual Studio 2022

Visual Studio コマンド ライン

使用: vs_enterprise.exe [command [options]]

(インストールする製品エディションに応じて vs_enterprise.exe を適宜置き換えてください)。

コマンド ライン コマンド 説明
(blank) 製品をインストールします。
変更 インストール済みの製品を変更します。
更新 インストール済みの製品を更新します。
修復 インストール済みの製品を修復します。
再開 インストールを再開します。
エクスポート インストール構成ファイルをエクスポートします。
modifySettings 既存のインストールの設定を変更します。
removeChannel チャネルを削除します。
updateAll 適用可能なすべてのインストールを更新します。
アンインストール インストール済みの製品をアンインストールします。


インストール コマンド ライン オプション 説明
--installPath <dir> インスタンスが処理するインストール ディレクトリです。 インストール コマンドの場合、これはオプションであり、インスタンスがインストールされる場所です。 他のコマンドの場合、これは必須であり、以前にインストールされたインスタンスがインストールされた場所です。
--addProductLang <language-locale> インストールまたは変更操作の間、これは製品にインストールされる UI 言語パックを決定します。 複数の言語パックを追加するために、コマンド ラインに複数回表示できます。存在しない場合、インストールではマシン ロケールを使用されます。 サポートされている言語ロケールの完全な一覧は以下を参照してください。
--removeProductLang <language-locale> インストールまたは変更操作の間、これは製品から削除される UI 言語パックを決定します。 複数の言語パックを追加するために、コマンド ラインに複数回表示できます。 サポートされている言語ロケールの完全な一覧は以下を参照してください。
--add < 1 つ以上のワークロードまたはコンポーネントを追加する IDs> 追加する 1 つ以上のワークロードまたはコンポーネント ID。 成果物の必須コンポーネントはインストールされますが、推奨コンポーネントまたはオプション コンポーネントはインストールされません。 --includeRecommended または -includeOptional を使用して、追加のコンポーネントをグローバルに制御できます。 複数のワークロードまたはコンポーネントを含めるには、--add コマンドを繰り返します (例: --add Workload1 --add Workload2)。 より細かく制御する場合、;includeRecommended または ;includeOptional を ID に追加できます (例: --add Workload1;includeRecommended または --add Workload2;includeRecommended;includeOptional)。 このオプションは、必要に応じて繰り返すことができます。
--remove < 1 つ以上のワークロードまたはコンポーネントを削除する IDs> 削除する 1 つ以上のワークロードまたはコンポーネント ID。
--in <path> 応答ファイルに対する URI またはパス。
--all 製品のすべてのワークロードとコンポーネントをインストールするかどうか。
--allWorkloads すべてのワークロードとコンポーネントをインストールしますが、推奨またはオプションのコンポーネントはインストールされません。
--includeRecommended インストールされているワークロードのコマンド済みコンポーネントが含まれますが、オプションのコンポーネントは含まれません。 ワークロードは、--allWorkloads または -add のいずれかで指定されます。
--includeOptional インストールされているワークロードのオプション コンポーネントが含まれますが、推奨コンポーネントは含まれません。 ワークロードは、--allWorkloads または -add のいずれかで指定されます。
--quiet, -q インストール実行中は、ユーザー インターフェイスを表示しないでください。
--passive, -p ユーザー インターフェイスを表示しますが、ユーザーからの対話は要求されません。
--norestart 存在する場合、--passive または --quiet を持つコマンドは (必要な場合は) 自動的にマシンを再起動することはありません。 --passive と --quiet のどちらも指定されていない場合、これは無視されます。
--nickname <name> インストール済み製品に割り当てるニックネームを定義します。 ニックネームは 10 文字以下にする必要があります。
--productKey <key> これは、インストールされている製品に使用するプロダクト キーを定義します。 これは、xxxxx-xxxxx-xxxxx-xxxxx-xxxxx または xxxxxxxxxxxxxxxxxxxxxxxxx の形式で 25 文字の英数字で構成されています。
--config <fullpath> インストールまたは変更操作の間、事前に保存されたインストール設定ファイルに基づいて、追加するワークロードとコンポーネント が決定されます。この操作は追加型であり、ファイル内にワークロードやコンポーネントが存在しない場合 削除されることはありません。また、製品に適用されない項目は追加されません。エクスポート操作の間、 インストール設定ファイルを保存する場所を決定します。 これは絶対パスである必要があります。
--help, --?, -h, -? パラメーターの使用状況を表示します。


レイアウト コマンド ライン オプション 説明
--layout <dir> ディレクトリを指定して、オフライン インストール キャッシュを作成します。
--lang <1 つ以上の言語ロケール> --layout と共に使用し、指定された言語のリソース パッケージを含むオフライン インストール キャッシュを準備します。 サポートされる言語ロケールの完全な一覧は以下を参照してください。
--add < 1 つ以上のワークロードまたはコンポーネントを追加する IDs> 追加する 1 つ以上のワークロードまたはコンポーネント ID。成果物の必須コンポーネントはインストールされますが、推奨コンポーネントまたはオプション コンポーネントはインストールされません。 --includeRecommended または -includeOptional を使用して、追加のコンポーネントをグローバルに制御できます。 より細かく制御する場合、;includeRecommended または ;includeOptional を ID に追加できます (例: --add Workload1;includeRecommended または --add Workload2;includeOptional)。
注意: --add を使用すると、指定されたワークロードとコンポーネントとその依存関係のみがダウンロードされます。 --add が指定されていない場合、すべてのワークロードとコンポーネントがレイアウトにダウンロードされます。
--includeRecommended インストールされているワークロードのコマンド済みコンポーネントが含まれますが、オプションのコンポーネントは含まれません。 ワークロードは、--allWorkloads または -add のいずれかで指定されます。
--includeOptional レイアウトに含まれるワークロード向けの推奨 と オプション コンポーネントが含まれます。 ワークロードは、-add で指定されます。
--keepLayoutVersion レイアウトのバージョンを更新せずに、変更を適用できます。
--verify レイアウトの内容を確認します。破損しているファイルまたは欠落しているファイルが一覧表示されます。
--fix レイアウトの内容を確認します。 ファイルが破損しているか見つからない場合は、再ダウンロードされます。 レイアウトを修正するには、インターネット接続が必要です。
--clean < カタログに対する 1 つ以上のパスをクリアする IDs> 新しいバージョンに更新されたレイアウトから以前のバージョンのコンポーネントを削除します。


詳細なコマンド ライン オプション 説明
--channelId <id> インストールするインスタンスのチャネルの ID。 これはインストール コマンドに必要で、--installPath が指定されている場合、他のコマンドでは無視されます。
--channelUri <uri> チャネル マニフェストの URI。更新が必要ない場合、--channelUri は存在しないファイルをポイントすることができます。 (例: --channelUri C:\doesntExist.chman) これはインストール コマンドに使用でき、他のコマンドでは無視されます。
--installChannelUri <uri> インストールに使用するチャネル マニフェストの URI。--channelUri で指定された URI (--installChannelUri が指定された場合に指定する必要があります) は、更新プログラムを検出するために使用されます。 これは、インストール コマンドに使用でき、他のコマンドでは無視されます。
--installCatalogUri <uri> インストールに使用するカタログ マニフェストの URI。 指定した場合、チャネル マネージャーは、インストール チャネル マニフェストで URI を使用する前に、この URI からカタログ マニフェストをダウンロードしようとします。 このパラメーターは、オフライン インストールをサポートするために使用され、既にダウンロード済みの製品カタログを使用してレイアウト キャッシュが作成されます。 これは、インストール コマンドに使用でき、他のコマンドでは無視されます。
--productId <id> インストールされるインスタンスの製品の ID。これは通常のインストール条件では事前入力されています。
--newChannelUri <uri> 更新に使用するチャネル マニフェストへの新しい URI またはパス。
--layoutUri <uri> インストール パッケージをフェッチするために使用される Web ホステッド レイアウト URI。
--theme <theme> インストーラーの色のテーマを変更します。値には Light/Dark/HighContrast を指定できます。
--wait プロセスは、インストールが完了するまで待機し、終了コードを返します。 これは、インストールからのリターン コードを処理するためにインストールが完了するまで待機する必要があるインストールを自動化する場合に役立ちます。
--locale <language-locale> インストーラー自体のユーザー インターフェイスの表示言語を変更します。設定は保持されます。 サポートされている言語ロケールの完全な一覧については、以下を参照してください。
--cache パッケージが存在する場合、インストール後に以降の修復のために保持されます。 これにより、後続のインストール、修復、または変更に使用されるグローバル ポリシー設定がオーバーライドされます。 既定のポリシーでは、パッケージをキャッシュします。これは、アンインストール コマンドでは無視されます。
--nocache 存在する場合、パッケージはインストールまたは修復された後に削除されます。 必要な場合にのみ再ダウンロードされ、使用後に再度削除されます。 これにより、後続のインストール、修復、または変更に使用されるグローバル ポリシー設定がオーバーライドされます。 既定のポリシーでは、パッケージをキャッシュします。
--noUpdateInstaller 存在する場合、quiet が指定されたときにインストーラーがそれ自体を更新できないようにします。 インストーラーの更新が必要な場合に、noUpdateInstaller が quiet で指定されていると、インストーラーはコマンドに失敗し、ゼロ以外の終了コードを返します。
--noWeb 存在する場合、Visual Studio セットアップでは、レイアウト ディレクトリ内のファイルを使用して Visual Studio をインストールします。 ユーザーがレイアウトにないコンポーネントをインストールしようとすると、セットアップは失敗します。
--path <name>=<path> インストール用のカスタム インストール パスを指定するために使用されます。サポートされているパス名は、shared、cache、install です。
--path cache=<path> 指定した場所を使用して、インストール ファイルをダウンロードします。この場所は、Visual Studio の初回インストール時のみ設定できます。 例: --path cache="C:\VS\cache"
--path shared=<path> Visual Studio を左右に並べてインストールための共有ファイルが含まれます。 一部のツールと SDK は、このドライブ上の場所にインストールされますが、他のツールではこの設定を上書きして別のドライブにインストールする場合があります。 例: --path shared="C:\VS\shared"

重要: この設定は、Visual Studio の初回インストール時に 1 回だけ設定できます。
--path install=<path> –-installPath と同等です。具体的には、--installPath "C:\VS" と --path install="C:\VS" は同等です。 一度に使用できるのは、いずれか 1 つだけです。
