Microsoft Visual Studio 2022

Visual Studio 命令列

使用方式: vs_enterprise.exe [command [options]]

(針對您要安裝的產品版本適當取代 vs_enterprise.exe。)

命令列命令 描述
(blank) 安裝產品。
modify 修改已安裝的產品。
update 更新已安裝的產品。
repair 修復已安裝的產品。
resume 繼續安裝。
export 匯出安裝組態檔。
modifySettings 修改現有安裝的設定。
removeChannel 移除通道。
updateAll 更新所有適用的安裝。
uninstall 解除安裝已安裝的產品。


安裝命令列選項 描述
--installPath <dir> 執行個體要執行的安裝目錄。 對於安裝命令而言，這是選擇性的，而且是安裝執行個體的位置。 對於其他命令，這是必要項，而且是先前安裝執行個體的位置。
--addProductLang <language-locale> 在安裝或修改作業期間，這會決定要安裝到產品的 UI 語言套件。 命令列上可能會出現多次，以新增多個語言套件。如果沒有，安裝會使用電腦的地區設定。 請參閱下方支援的語言地區設定完整清單。
--removeProductLang <language-locale> 在安裝或修改作業期間，這會決定要從產品移除的 UI 語言套件。 命令列上可能會出現多次，以新增多個語言套件。 請參閱下方支援的語言地區設定完整清單。
--add <一或多個工作負載或元件識別碼> 要新增的一或多個工作負載或元件識別碼。 成品的必要元件已安裝，但不是建議的元件或選擇性元件。 您可以使用 --includeRecommended 和/或 --includeOptional，全域控制其他元件。 若要包含多個工作負載或元件，請重複 --add 命令 (例如，--add Workload1 --add Workload2)。 若是更精細的控制項，您可以將;includeRecommended 或 ;includeOptional 附加至識別碼 (例如，--add Workload1;includeRecommended 或 --add Workload2;includeRecommended;includeOptional)。 您可以在必要時重複此選項。
--remove <一或多個工作負載或元件識別碼> 要移除的一或多個工作負載或元件識別碼。
--in <path> 回應檔案的 URI 或路徑。
--all 是否要安裝產品的所有工作負載與元件。
--allWorkloads 安裝所有工作負載和元件，沒有建議或選擇性的元件。
--includeRecommended 包含已安裝之任何工作負載的建議元件，但不包含選擇性元件。 工作負載是以 --allWorkloads 或 --add 來指定。
--includeOptional 包含已安裝之任何工作負載的選擇性元件，但不包含建議的元件。 工作負載是以 --allWorkloads 或 --add 來指定。
--quiet, -q 執行安裝時不要顯示任何使用者介面。
--passive, -p 顯示使用者介面，但不要求使用者進行任何互動。
--norestart 如果有，命令會使用 --passive 或 --quiet 必要時不會自動重新啟動電腦。 如果未指定--passive 或-quiet，則會略過此問題。
--nickname <name> 這會定義要指派給已安裝產品的暱稱。 暱稱不能超過 10 個字元。
--productKey <key> 這會定義要用於已安裝產品的產品金鑰。 它由 25 個英數字元組成，格式為 xxxxx-xxxxx-xxxxx-xxxxx-xxxxx 或 xxxxxxxxxxxxxxxxxxxxxxxxx。
--config <fullpath> 在安裝或修改作業期間，這會決定要新增的工作負載與元件 根據先前儲存的安裝設定檔。此作業為加總量值，不會移除任何工作負載或元件 如果檔案中沒有這些檔案。此外，將不會新增不適用於產品的項目。在匯出作業期間， 這會決定儲存安裝設定檔的位置。 這必須是絕對路徑。
--help, --?, -h, -? 顯示參數使用方式。


版面配置命令列選項 描述
--layout <dir> 指定目錄以建立離線安裝快取。
--lang <one or more language-locales> 與 --layout 一起使用，以準備具有指定語言之資源套件的離線安裝快取。 請參閱下方支援的語言地區設定完整清單。
--add <一或多個工作負載或元件識別碼> 要新增的一或多個工作負載或元件識別碼。成品的必要元件已安裝，但不是建議的元件或選擇性元件。 您可以使用 --includeRecommended 和/或 --includeOptional，全域控制其他元件。 若是更精細的控制項，您可以將;includeRecommended 或 ;includeOptional 附加至識別碼 (例如，--add Workload1;includeRecommended 或 --add Workload2;includeOptional)。
注意: 如果使用 --add，則只會下載指定的工作負載和元件及其相依性。 如果未指定 --add，則會將所有工作負載和元件下載到版面配置。
--includeRecommended 包含已安裝之任何工作負載的建議元件，但不包含選擇性元件。 工作負載是以 --allWorkloads 或 --add 來指定。
--includeOptional 包含建議的和配置中所包含之任何工作負載的選擇性元件。 工作負載是以 --add 指定。
--keepLayoutVersion 在不更新版面配置版本的情況下，將變更套用至版面配置。
--verify 確認版面配置的內容。列出任何損毀或遺失的檔案。
--fix 確認版面配置的內容。 如果發現任何檔案已損毀或遺失，則會重新下載。 需要網際網路存取才能修正版面配置。
--clean <一或多個目錄路徑> 從已更新為較新版本的版面配置移除舊版元件。


進階命令列選項 描述
--channelId <id> 要安裝之執行個體的通道識別碼。 安裝命令需要此命令，如果指定了 --installPath，則會略過其他命令。
--channelUri <uri> 通道資訊清單的 URI。如果不需要更新，--channelUri 可以指向不存在的檔案。 (例如 --channelUri C:\doesntExist.chman) 這可以用於安裝命令; 其他命令會略過它。
--installChannelUri <uri> 用於安裝的通道資訊清單 URI。--channelUri 指定的 URI (用來偵測更新時，必須指定 --installChannelUri)。 這可以用於安裝命令; 其他命令會略過它。
--installCatalogUri <uri> 要用於安裝之目錄資訊清單的 URI。 若指定，頻道管理員會嘗試從此 URI 下載類別目錄資訊清單，然後再使用安裝通道資訊清單中的 URI。 此參數用於支援離線安裝，其中配置快取會使用已下載的產品目錄建立。 這可以用於安裝命令; 其他命令會略過它。
--productId <id> 即將安裝之執行個體的產品識別碼。這會在一般安裝條件中預先填入。
--newChannelUri <uri> 用於更新之通道資訊清單的新 URI 或路徑。
--layoutUri <uri> 用來擷取安裝套件的 Web 主控配置 URI。
--theme <theme> 變更安裝程式的色彩佈景主題。值可以是 Light/Dark/HighContrast。
--wait 處理序將等候安裝完成，再傳回結束代碼。 當需要等候安裝完成才能處理該安裝傳回碼的安裝時，此功能很有用。
--locale <language-locale> 變更安裝程式本身使用者介面的顯示語言。設定將保存。 請參閱下方支援的語言地區設定完整清單。
--cache 如果有，套件會在安裝後保留以進行後續修復。 這會覆寫要用於後續安裝、修復或修改的全域原則設定。 預設原則是快取封裝。解除安裝命令會略過此動作。
--nocache 如果存在，套件將會在安裝或修復後刪除。 只有在需要時才會再次下載，並在使用後再次刪除。 這會覆寫要用於後續安裝、修復或修改的全域原則設定。 預設原則是快取封裝。
--noUpdateInstaller 如果有，則會在指定 quiet 時防止安裝程式自行更新。 如果在需要安裝程式更新時以安靜模式指定 noUpdateInstaller，則安裝程式會讓命令失敗，並傳回非零的結束代碼。
--noWeb 如果有，Visual Studio 安裝程式會使用配置目錄中的檔案來安裝Visual Studio。 如果使用者嘗試安裝不在配置中的元件，安裝程式會失敗。
--path <name>=<path> 用來指定安裝的自訂安裝路徑。支援的路徑名稱為共用、快取及安裝。
--path cache=<path> 使用您指定的位置來下載安裝檔案。此位置只能在初次安裝 Visual Studio 時設定。 範例: --path cache="C:\VS\cache"
--path shared=<path> 包含用於並排 Visual Studio 安裝的共用檔案。 某些工具和 SDK 會安裝到此磁碟機上的位置，而其他工具和 SDK 可能會覆寫此設定並安裝到其他磁碟機。 範例: --path shared="C:\VS\shared"

重要: 這只能在安裝 Visual Studio 時或首次設定一次。
--path install=<path> 相當於 –-installPath。具體來說，--installPath "C:\VS" 和 --path install="C:\VS" 相等。 一次只能使用其中一項。
